2. Требования к системе
2.1 Требования к функциональности:
2.1.1. Регистрация и аутентификация пользователей:
Пользователь должен иметь возможность зарегистрироваться в системе, указав логин, пароль и адрес электронной почты.
Система должна обеспечивать надежную аутентификацию пользователей при каждом входе в систему.
2.1.2. Возможность читать статьи:
Сайт должен корректно отображать все статьи.
У пользователя должна быть возможность прочитать статью полностью. 
2.1.3. Прохождение тестов:
Сайт должен корректно отображать все тесты.
Пользователь должен иметь возможность пройти любой тест.
После прохождения теста пользователю должны будут показываться баллы успеха прохождения теста.
После прохождения теста должна быть возможность посмотреть правильные и не правильные ответы.
2.1.4. Администрирование (дополнительно):
Администратор должен иметь возможность добавлять, удалять и редактировать тесты.
Администратор должен иметь возможность добавлять, удалять и редактировать статьи.
Это всё может быть реализованно с помощью панели администратора.
2.2 Требования к пользовательскому интерфейсу (UI):
2.2.1. Общие требования:
Интерфейс должен быть интуитивно понятным и удобным для использования.
Дизайн должен быть современным и привлекательным.
Сайт должен отображаться только на компьютерах
Интерфейс должен быть переведен на русский язык.
Основная палитра должна состоять из 4 цветов.
2.2.2. Начальная страница:
Кнопка входа/регистрации пользователя.
Кнопка перехода на странцу с информацией о сайте.
2.2.3. Страница Про нас:
Информация о сайте.
Кнопка перехода на начальную страницу.
Кнопка на страницу регистрации.
2.2.4. Страница регистрации:
Поля ввода логина, электронной почты, пароля и повтора пароля.
Кнопка регестрации.
Кнопка перехода на начальную страницу.
Кнопка перехода на страницу про нас.
2.2.5. Страница авторизации:
Поля ввода логина и пароля.
Кнопка входа.
Кнопка перехода на начальную страницу.
2.2.6. Домашняя страница:
Навигационная панель слева сбоку.
Возможность перехода на другие страницы с помощью навигационной панели.
Весь контент отдельной страницы должен отображаться справа сбоку от навигационной панели.
В самом верху страницы отображение блока с названием сайта.
2.2.7. Страница со статьями:
Навигационная панель слева сбоку.
Каждая статья - отдельный блок выделенный цветом.
Кнопка читать далее.
2.2.8. Страница с тестами:
Навигационная панель слева сбоку.
Каждый тест - отдельный блок выделенный цветом.
Название теста.
Кнопка пройти тест.
2.2.9. Страница выхода из аккаунта:
Кнопка выхода.
Кнопка возвращения на домашнюю страницу.
2.3 Требования к производительности:
Время отклика системы на запрос пользователя не должно превышать 2 секунды.
2.4 Требования к надежности:
Сайт должен сохранять все данные пользователя на сайте в отдельной бд.
2.5 Требования к безопасности:
Система должна обеспечивать защиту пользовательских данных от несанкционированного доступа.
Система должна использовать надежные алгоритмы шифрования для хранения паролей пользователей.

3. Описание программного обеспечения
3.1 Архитектура системы:
Система будет иметь трехуровневую архитектуру:
oУровень представления (Presentation Layer): Пользовательский интерфейс, реализованный на HTML, CSS и JavaScript.
oУровень бизнес-логики (Business Logic Layer): обрабатывает ответы пользователя на тесты, взаимодействует с базой данных.
oУровень данных (Data Access Layer): Отвечает за хранение и извлечение данных из базы данных.
3.2 Описание основных модулей:
Модуль аутентификации: Отвечает за регистрацию, аутентификацию и управление учетными записями пользователей.
Модуль тестов: Отвечает за хранение прогресса пользователя в тестах.
Модуль администрирования (дополнительно): Предоставляет администратору интерфейс для управления системой.
3.3 Описание структур данных:
Таблица пользователей: Содержит информацию о пользователях (ID, логин, пароль, email).
